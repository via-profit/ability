## Что дают сгенерированные типы

Генератор типов парсит политики и создает типы, которые следует передавать в дженерики классов для того, чтобы IDE линтер подсказывал при ошибках и опечатках.
Это необязательно, поэтому если лень запариваться за генерацию типов, можно это пропустить.

Полученные типы можно использовать для строгой типизации ресурсов при вызове `resolver.resolve()` и `resolver.enforce()`.

### Что генерируется

| Тип | Назначение | Пример использования |
|-----|------------|---------------------|
| **`Resources`** | Тип всех доступных ресурсов (субъектов) в DSL | `document.*`, `order.*`, `user.*` |
| **`Environment`** | Тип данных окружения в DSL | `env.time.*`, `env.ip.*`, `env.user.*` |
| **`PolicyTags`** | Union-тип всех тегов политик | `"admin" \| "user" \| "moderator"` |

## Генерация типов из политик


```ts
import { AbilityTypeGenerator } from '@via-profit/ability';
import { policies } from './policies';

// Создаем генератор и генерируем типы
const generator = new AbilityTypeGenerator(policies);
const typeDefs = generator.generateTypeDefs();

// Сохраняем в файл
fs.writeFileSync('ability.types.ts', typeDefs, { encoding: 'utf-8' });
```

## Использование сгенерированных типов

После генерации типов вы можете использовать их в `AbilityDSLParser` и `AbilityResolver`, а так же во всех классах, где есть соответствующий дженерик:

```typescript
import { AbilityDSLParser, AbilityResolver, DenyOverridesStrategy } from '@via-profit/ability';
import type { Resources, Environment, PolicyTags } from './ability.types';

// DSL с типизацией
const dsl = `
  @name "Чтение документов"
  permit permission.document.read if all:
    document.ownerId equals user.id
    document.status in ["published", "archived"]
`;

// Парсим DSL с типизацией
const policies = new AbilityDSLParser<Resources, Environment, PolicyTags>(dsl).parse();

// Создаем резолвер с типами
const resolver = new AbilityResolver(policies, DenyOverridesStrategy);

// Используем с автодополнением и проверкой типов
resolver.enforce('document.read', {
  document: {
    ownerId: 'user-123', // ✅ string
    status: 'published'   // ✅ "published" | "archived"
  }
}, {
  time: {
    hour: 14,   // ✅ number
    minute: 30  // ✅ number
  }
});

// ❌ Ошибка компиляции: ownerId должен быть строкой
resolver.enforce('document.read', {
  document: {
    ownerId: 22, // ❌ Type error. Number is not a string'
    status: 'archived' 
  }
}, {
  time: {
    hour: 14,
    minute: 30
  }
});
```

## Интеграция с DSL

Для удобства работы с DSL, пакет предоставляет шаблонную функцию `ability`:

```ts
import { ability } from '@via-profit/ability';
import type { Resources, Environment, PolicyTags } from './ability.types';

// DSL с полной типизацией и автодополнением
const policies = ability<Resources, Environment, PolicyTags>`
  @name "Управление заказами"
  permit permission.orders.read if any:
    order.author.id equals user.id
    env.time.hour less than 18

  @name "Администрирование"
  @tags ["admin"]
  permit permission.orders.* if all:
    user.role equals "admin"
`;

const resolver = new AbilityResolver(policies, DenyOverridesStrategy);
```

> [!TIP]
> Функция `ability` — это тегированный шаблон, который автоматически парсит DSL и возвращает массив политик с правильными типами.

## Автоматизация генерации

Добавьте в `package.json`:

```json
{
  "scripts": {
    "generate:ability": "node scripts/generate-ability-types.js"
  }
}
```

```ts
// scripts/generate-ability-types.js
const fs = require('node:fs');
const path = require('node:path');
const { AbilityDSLParser, AbilityTypeGenerator } = require('@via-profit/ability');

const dslPath = path.resolve(__dirname, '../src/ability');
const outputFile = path.resolve(__dirname, '../src/ability/ability.types.ts');

// Собираем все DSL файлы
const dslFiles = fs.readdirSync(dslPath).filter(f => f.endsWith('.dsl'));
let dsl = '';
for (const file of dslFiles) {
  dsl += fs.readFileSync(path.join(dslPath, file), 'utf-8') + '\n';
}

// Парсим и генерируем типы
const policies = new AbilityDSLParser(dsl).parse();
const typeDefs = new AbilityTypeGenerator(policies).generateTypeDefs();

fs.writeFileSync(outputFile, typeDefs, 'utf-8');
console.log('✅ Types generated successfully!');
```

## Частые вопросы

**Нужно ли генерировать типы каждый раз при изменении политик?**
Да, типы должны быть синхронизированы с политиками. Рекомендуется добавить автоматическую генерацию в pre-commit хуки или watch-режим.

**Как обновить типы на клиенте?**
Если вы используете HTTP-сервер артефактов, клиент может загружать свежие типы по запросу. Или просто скопируйте сгенерированный файл в клиентский проект.
