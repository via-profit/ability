# Roadmap

## 1. Ближайшие задачи (высокий приоритет)

### 1.1. Кэширование результатов политик
**Цель:** ускорить работу при большом количестве политик.  
**Описание:**  
- Кэшировать результаты `policy.check(resource)` на время одного запроса.  
- Кэшировать результаты wildcard‑сопоставления действий.  
- Возможность отключать кэш.

### 1.2. Улучшенная валидация политик
**Цель:** предотвращать ошибки на этапе загрузки.  
**Описание:**  
- Проверка корректности `subject` и `resource` путей.  
- Проверка конфликтующих правил.  
- Проверка дублирующихся `action`.  
- Проверка пустых ruleSet.

### 1.3. Улучшение AbilityExplain
**Цель:** сделать объяснения более полезными.  
**Описание:**  
- Добавить уровни детализации (короткое / полное объяснение).  
- Добавить структурированный JSON‑формат.  
- Добавить возможность трассировки всех правил, включая пропущенные.

### 1.4. Улучшение документации
**Цель:** сделать библиотеку понятной для новых пользователей.  
**Описание:**  
- Добавить разделы Recipes (Express, NestJS, GraphQL).  
- Добавить примеры тестирования политик.  
- Добавить FAQ.

---

## 2. Среднесрочные задачи (средний приоритет)

### 2.1. Интеграции

#### Express.js
- Middleware (уже есть рецепт, нужно оформить как пакет).

#### NestJS
- Guard + Decorator (`@Ability('order.update')`).

#### Fastify
- Плагин с хуками.

#### Apollo GraphQL
- Директива (`@ability(action: "order.update")`).

### 2.2. AbilityParser: динамические атрибуты
**Цель:** расширить возможности ABAC.  
**Описание:**  
Поддержка функций в subject/resource:

```ts
subject: (ctx) => ctx.user.id
```

### 2.3. AbilityParser: generateTypeDefs v2
**Цель:** улучшить генерацию типов.  
**Описание:**  
- Генерация комментариев JSDoc.  
- Генерация union‑типов для action.  
- Генерация типов для Explain.

### 2.4. CLI‑утилита
**Цель:** автоматизация.  
**Описание:**  
- Генерация типов.  
- Валидация политик.  
- Форматирование JSON‑политик.  
- Проверка конфликтов.

---

## 3. Долгосрочные задачи (низкий приоритет)

### 3.1. Визуальный инспектор политик
**Цель:** удобная отладка.  
**Описание:**  
- Веб‑интерфейс для просмотра политик.  
- Визуализация ruleSet.  
- Возможность тестировать ресурс вручную.

### 3.2. Редактор политик
**Цель:** управление политиками без правки JSON.  
**Описание:**  
- UI для создания правил.  
- Экспорт в JSON.  
- Импорт из JSON.

### 3.3. Поддержка environment‑атрибутов
**Цель:** полноценный ABAC.  
**Описание:**  
- Время суток.  
- Геолокация.  
- IP‑адрес.  
- Контекст запроса.

### 3.4. Поддержка асинхронных правил
**Цель:** расширить возможности.  
**Описание:**  
- Возможность проверять внешние источники (БД, API).  
- Асинхронный `check()`.

---

## 4. Исследовательские задачи (экспериментальные)

### 4.1. Компилятор политик
**Цель:** максимальная производительность.  
**Описание:**  
- Компиляция политик в JavaScript‑функции.  
- Ускорение выполнения в 10–50 раз.

### 4.2. Поддержка WebAssembly
**Цель:** выполнение политик в sandbox.  
**Описание:**  
- Компиляция правил в WASM.  
- Изоляция выполнения.

### 4.3. Интеграция с Open Policy Agent (OPA)
**Цель:** гибридный режим.  
**Описание:**  
- Возможность использовать OPA для сложных политик.  
- AbilityResolver как слой над OPA.

---

## 5. Улучшение DX (Developer Experience)

### 5.1. Улучшенные ошибки
**Цель:** сделать библиотеку дружелюбнее.  
**Описание:**  
- Чёткие сообщения об ошибках.  
- Подсказки по исправлению.  
- Ссылки на документацию.

### 5.2. Улучшение логирования
**Цель:** удобная отладка.  
**Описание:**  
- Логирование совпавших политик.  
- Логирование пропущенных политик.  
- Логирование wildcard‑совпадений.

### 5.3. Улучшение тестирования
**Цель:** удобные тесты.  
**Описание:**  
Добавить утилиты:

```ts
expectAllow('order.update', resource);
expectDeny('order.update', resource);
```
